# Performance: Icon Migration + SW Caching Design

## Goal

Eliminate ~280KB of font download on first load by replacing the MDI icon font with inline SVG icons, and add service-worker caching so the home screen renders from cache instantly on every reopen — with immediate cache invalidation on writes and socket events so users always see fresh data.

## Architecture

Two independent changes with no shared code:

1. **Icon migration** — `@mdi/font` (CSS font, all 7000+ icons) → `@mdi/js` (tree-shaken SVG paths, only the 205 icons actually used). A single `AppIcon.vue` component replaces all `<v-icon>` usages.

2. **SW caching + invalidation** — Workbox RuntimeCaching in `vite.config.ts` caches cover images and key API reads. A `cache.ts` utility provides explicit invalidation called from mutation sites and socket event handlers.

## Tech Stack

- `@mdi/js` (already a transitive dep of Vuetify, just not used directly)
- Workbox (already active via `vite-plugin-pwa`)
- Cache Storage API (browser-native, no extra deps)

## Part 1: Icon Migration

### File Structure

- **Create** `src/utils/icons.ts` — re-exports the 205 used MDI icon paths from `@mdi/js` as named exports
- **Create** `src/components/common/AppIcon.vue` — renders an inline `<svg>` using a path looked up from `icons.ts`
- **Modify** `src/plugins/vuetify.ts` — remove `import '@mdi/font/css/materialdesignicons.css'`
- **Modify** all `*.vue` files — replace `<v-icon ...>mdi-x</v-icon>` with `<AppIcon icon="mdi-x" ... />`

### AppIcon.vue

Accepts `icon` (kebab string e.g. `"mdi-close"`), `size` (number, default 24), `color` (string, default `"currentColor"`). Converts icon name to camelCase key (`"mdi-close"` → `"mdiClose"`), looks up the SVG path from `icons.ts`, renders a `<svg viewBox="0 0 24 24">` with a single `<path fill="currentColor">`. Passes through `class` and `style` attributes so existing animation classes (`.spin`) continue working.

```vue
<template>
  <svg
    :width="size"
    :height="size"
    viewBox="0 0 24 24"
    class="app-icon"
    :style="color && color !== 'currentColor' ? `color: ${color}` : undefined"
    aria-hidden="true"
  >
    <path :d="path" fill="currentColor" />
  </svg>
</template>

<script setup lang="ts">
import { computed } from 'vue'
import * as icons from '@/utils/icons'

const props = defineProps<{
  icon: string
  size?: number | string
  color?: string
}>()

const path = computed(() => {
  const key = 'mdi' + props.icon
    .replace(/^mdi-/, '')
    .split('-')
    .map(w => w.charAt(0).toUpperCase() + w.slice(1))
    .join('')
  return (icons as Record<string, string>)[key] ?? ''
})
</script>

<style scoped>
.app-icon { display: inline-block; vertical-align: middle; flex-shrink: 0; }
</style>
```

### icons.ts

Re-exports only the 205 icons used across the codebase:

The full export list is generated by running `grep -roh "mdi-[a-z0-9-]*" src/ --include="*.vue" --include="*.ts" | sort -u`, converting each `mdi-foo-bar` to `mdiFooBar`, and adding an export for each. The implementation task includes this generation step explicitly.

### Migration Script

A one-shot bash+sed script transforms all `<v-icon>` usages:

```bash
# Replace <v-icon size="N" color="C">mdi-x</v-icon>
# with    <AppIcon icon="mdi-x" :size="N" color="C" />
```

Handled patterns:
- Static size: `size="16"` → `:size="16"`
- Dynamic size: `:size="foo"` → `:size="foo"` (unchanged)
- Color prop: preserved as-is
- Class prop (e.g. `class="spin"`): preserved as-is
- Multiline `<v-icon>`: collapsed to single-line `<AppIcon>`
- Import of `AppIcon` added to each modified file's `<script setup>`

After migration, run `npx vue-tsc --noEmit` to catch any missed edge cases.

## Part 2: SW Caching + Invalidation

### File Structure

- **Modify** `vite.config.ts` — add `runtimeCaching` to Workbox config
- **Create** `src/utils/cache.ts` — `invalidateCovers(itemId)`, `invalidateApiEntries(pattern)` 
- **Modify** `src/stores/socket.ts` — call cache invalidation on library socket events
- **Modify** `src/components/sheets/MatchSheet.vue` — call invalidation after `doApply`
- **Modify** `src/components/sheets/BookDetailSheet.vue` — call invalidation after `doSaveMeta`, `doUpdateCover`
- **Modify** any other mutation sites (CoverSheet, delete item)

### Workbox RuntimeCaching Config

```ts
runtimeCaching: [
  {
    // Cover images — fast reads, explicit invalidation on write
    urlPattern: /\/api\/items\/[^/]+\/cover/,
    handler: 'CacheFirst',
    options: {
      cacheName: 'covers-cache',
      expiration: { maxEntries: 500, maxAgeSeconds: 7 * 24 * 60 * 60 },
      cacheableResponse: { statuses: [0, 200] },
    },
  },
  {
    // Key API reads — instant render, background refresh
    urlPattern: /\/api\/(me\/items-in-progress|me$|libraries\/[^/]+\/items|me\/progress)/,
    handler: 'StaleWhileRevalidate',
    options: {
      cacheName: 'api-cache',
      expiration: { maxEntries: 100, maxAgeSeconds: 24 * 60 * 60 },
      cacheableResponse: { statuses: [0, 200] },
    },
  },
],
```

Only GET requests are cached (Workbox default). PATCH/POST/DELETE always go to the network.

### cache.ts

```ts
const COVERS_CACHE = 'covers-cache'
const API_CACHE    = 'api-cache'

export async function invalidateCovers(itemId: string): Promise<void> {
  if (!('caches' in window)) return
  const cache = await caches.open(COVERS_CACHE)
  const keys  = await cache.keys()
  await Promise.all(
    keys
      .filter(req => req.url.includes(`/items/${itemId}/cover`))
      .map(req => cache.delete(req))
  )
}

export async function invalidateApiEntries(pattern: string): Promise<void> {
  if (!('caches' in window)) return
  const cache = await caches.open(API_CACHE)
  const keys  = await cache.keys()
  await Promise.all(
    keys
      .filter(req => req.url.includes(pattern))
      .map(req => cache.delete(req))
  )
}
```

Both functions are fire-and-forget (callers do not await them unless needed).

### Invalidation Callsites

| File | Function | Invalidation call |
|---|---|---|
| `MatchSheet.vue` | `doApply` (after getItem) | `invalidateCovers(itemId)`, `invalidateApiEntries(itemId)` |
| `BookDetailSheet.vue` | `doSaveMeta` (after getItem) | `invalidateApiEntries(itemId)` |
| `BookDetailSheet.vue` | `doUpdateCover` (after POST) | `invalidateCovers(itemId)` |
| `BookDetailSheet.vue` | `doDeleteItem` (after DELETE) | `invalidateApiEntries(libraryId)` |
| `CoverSheet.vue` | after successful cover save | `invalidateCovers(itemId)` |
| `socket.ts` | `library_item_added` | `invalidateApiEntries(libraryId)` |
| `socket.ts` | `library_item_updated` | `invalidateApiEntries(itemId)` |
| `socket.ts` | `library_item_removed` | `invalidateApiEntries(libraryId)` |
| `socket.ts` | `library_scan_complete` | `invalidateApiEntries(libraryId)` |
